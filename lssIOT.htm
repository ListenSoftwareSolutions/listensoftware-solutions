 Process

The data gathering process begins with the state machine. A state machine made possible through an Android operating system runs on a device. The state machine can do many things at the same time: it can display data to the user, it can gather data from the user, and it can gather data from other devices either by bluetooth, sockets and pipes, or serial communication. These methods of data communication are simple, they make possible, the transmission of data from one device to another.

The information is stored in large databases on the cloud. The information in the database can be used later to create dashboard, reports, and data extracts.

The galaxy tablet uses the android operating system to create a state machine where one or more applications can execute. The tablet application is written in the c# language. The c# language is used to communicate with the tablet and external devices. An external device can be connected to by bluetooth or socket. The device is then known by its machine address internally. The name of the external device and its machine address are used to communicate with it.

Data being transmitted from the external device is intercepted in by a serial communication function in the application. The data is converted to text and displayed on an user form for display. The user reviews the data and they presses a user interface button for transmission of the data to a server. The data is received on the server and then saved to a database table for storage.

The sheer number of transactions involved makes the cloud the right place to store the data for analytics. Database costs are always the inhibitor to large scale projects. Research methods for reduce database transactional cost is always the goal.

The next step is to build an analytics engine to help customers study and understand their data. Less than 25% of the people gathering information from things are studying the data. Analytics and Key Performance Indicators are helpful tools in understand what is happening in data.
