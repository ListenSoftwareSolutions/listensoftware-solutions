Devices

The Internet of Things (IoT) process begins with the machine. A state machine made possible through an operating system runs on a device. The state machine can do many things at the same time: it can display data to the user, gather data from the user and gather data from other devices, either by Bluetooth, sockets and pipes, or serial communication. The data is wrapped up in a data packet called a JavaScript Object Notation (JSON) and sent by HTTPS to a server web program. The web program then receives the data and stores it on large databases like Microsoft SQL Server.
The Galaxy tablet uses the Android operating system to create a state machine where one or more applications can execute. The tablet application is written in the Java programming language. Java is used to communicate between the tablet and external devices. An external device can be connected by Bluetooth or socket. The device is then known by its machine address internally. The name of the external device and its machine address are used to communicate with it. The serial object is then associated with the external device, and an input or output datastream is bound to the serial communication. Data being transmitted from the external device is intercepted in a call back function in the Java application. The data is converted to text and displayed on a user-friendly form. The user reviews the data, and he presses a button for transmission of the data to a server. A web server processes the HTTP post and calls the web API being requested. The data is received on the server and then saved to a database table for storage.
